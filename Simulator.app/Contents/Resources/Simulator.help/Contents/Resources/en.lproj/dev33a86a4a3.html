Test authentication

Use Simulator to test parts of your biometric authentication flow on devices that support authentication.

Configure the system authentication enrolled state

Test that your code correctly handles the error that occurs when the user is not enrolled for authentication using Touch ID or Face ID.

  • Choose Hardware > Touch ID/Face ID > Enrolled to change the current state.

    A checkmark by the menu item sets the system enrolled state to true for Touch ID or Face ID. When there is no checkmark the system returns an error indicating there are no fingers enrolled for Touch ID or faces enrolled for Face ID.

Simulate matching and non-matching authentications

Test responding to matching and non-matching authentications that use Touch ID or Face ID.

  • When the authentication dialog is presented, make a choice from Hardware > Touch ID/Face ID for a matching or non-matching touch or face.

For details on how to check for authentication, see the LocalAuthentication Framework API reference.
