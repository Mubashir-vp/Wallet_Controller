Insufficient resources error when trying to launch a simulator

Launching a simulator can result in an error alert that there are insufficient system resources. This usually occurs when launching a simulator exceeds either the maximum number of active processes or the maximum number of open files. The best solution is to free up resources by closing simulated devices and other Mac applications.

If it is not possible to free up enough resources, you can raise the system limits until the Mac is restarted.

Note: It is possible to exceed the maximum number of process or the maximum number of open files by launching other programs or opening other files some time after you have launched a simulator. If this occurs, Simulator does not give you a warning. The effects depend on what program is trying to launch the process or open the file.

Find the current process and file limits

  1. Open the terminal.

  2. Run the command sudo launchctl limit.

    You will need to type your password as the command is run as root.

  3. Note down the maximum number of processes shown next to maxproc, and the maximum number of files shown next to maxfiles.

Find the total number of running processes

  • Run the command ps -A | wc -l in the terminal. The number shown is two more than the total running processes. This is because wc is counting the header line and a blank line at the end of the listing.

Increase the maximum number of processes until the next restart

  • Run the command sudo launchctl limit maxproc <per user limit> <total limit>.

    For example, the command line:

    sudo launchctl limit maxproc 2000 2500

    sets the maximum number of concurrent processes for any one user to 2000 and the total number of all concurrent processes to 2500.

    WARNING: Setting the the maximum number of processes to a number that is too low can prevent your Mac from operating correctly. Restart your machine to restore the original limits.

Increase the maximum number of open files until the next restart

  • Run the command sudo launchctl limit maxfiles <per process limit> <total limit>.

    For example, the command line:

    sudo launchctl limit maxfiles 2000 unlimited

    sets the maximum number of open files for any one process to 2000 and the total number of all open files to unlimited.

    WARNING: Setting the the maximum number of open files per process to a number that is too low can prevent your Mac from operating correctly. Restart your machine to restore the original limits.
